# CTalkDiversity Conferences 1–23 이미지 감사 / 표준 매핑 제안

기준:
- `main` = 회차 대표 포스터/종합 이벤트 포스터
- `speaker-N` = 개별 발표자(또는 발표팀) 중심 이미지
- `extra-N` = 티저, 일정표, 종합 요약, 등록 안내 등 보조 이미지
- **실제 파일 변경은 하지 않음.** 기존 HTML 연결 위험 때문에 우선 정확한 매핑표만 확정.
- 1~18은 원본 숫자파일과 표준파일이 **바이트 동일(md5 일치)** 하며, 19~23은 표준파일만 존재.

## 요약

### 현재 상태가 대체로 맞는 회차
- 5, 12, 15, 16, 17, 18, 19, 20, 22

### 분류/이름 수정이 필요한 회차
- 1, 2, 3, 4, 6, 7, 8, 9, 10, 11, 13, 14, 21, 23

### 대표적인 수정 패턴
- `speaker`로 되어 있지만 실제론 `extra`인 경우: 1,2,3,4,6,7
- `extra`로 되어 있지만 실제론 `speaker-3`인 경우: 8,9,11,14
- `main`이 잘못 잡힌 경우: 8,10,13
- speaker 순서(포스터 표기 순서 vs 현재 파일순) 재검토 여지: 21,23

### 회차별 특이사항
- **1,2,3,4,7:** 개별 speaker 포스터가 거의 없고, 일정/티저/종합형 중심
- **5:** 이미지 1장뿐
- **8,9,10,11,14:** speaker 3명
- **12:** speaker 팀 1개 + 개인 1명
- **18:** speaker 1명
- **21,23:** speaker 번호 순서 해석 여지 있음

## 권장 다음 단계
1. 이 문서를 기준으로 HTML 참조 여부를 먼저 확인
2. HTML이 숫자 원본파일을 직접 참조하지 않으면,
   - 원본 숫자파일 제거 전 백업
   - 표준파일명을 위 권장안대로 재정렬
3. HTML이 이미 현재 표준파일명을 참조 중이면,
   - 실제 파일 rename 대신 HTML 쪽에서 canonical mapping 반영 여부 검토
   - 또는 alias/복제 유지 전략 사용
